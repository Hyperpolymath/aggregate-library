# Scope

# This security policy covers:
# - Specification documents and their semantics
# - Test cases and examples
# - Documentation accuracy
# - Repository infrastructure
#
# Security issues in language implementations should be reported
# to the respective implementation maintainers.

# Reporting Guidelines

# IMPORTANT: Do NOT open public issues for security vulnerabilities.
#
# To report security vulnerabilities:
# 1. Use GitHub Security Advisory (preferred)
# 2. Email security@[project-domain] when available
# 3. Provide: description, impact, affected components, reproduction steps
#
# Expected response times:
# - Initial response: 72 hours
# - Triage: 1 week
# - Critical fixes: Days to weeks depending on severity
#
# We follow coordinated disclosure practices.
